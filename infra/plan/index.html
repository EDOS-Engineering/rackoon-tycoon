Rackoon Tycoon — Deployment Plan

Production go-live on rackoon.cloud · static site · DigitalOcean droplet via Laravel Forge · CI deploy from GitHub Actions

Summary. Rackoon Tycoon is a 100% static site (study guide + canvas game, vanilla JS ES modules, zero runtime deps no build step). We serve the repository's static files directly from an existing DigitalOcean droplet that Laravel Forge already manages (nginx + Let's Encrypt). We point the new domain rackoon.cloud at the droplet's static IP, create a Forge Static HTML site connected to the GitHub repo, and let GitHub Actions drive deploys: on every push to main a verify job runs the dev-only Node sim harness as a gate, then a deploy job hits the Forge deploy webhook (which does a plain git pull on the droplet — no build), finishing with a post-deploy health check against https://rackoon.cloud. No backend, no bundler, no secrets in the repo.

Architecture

Flow. Developer pushes to main → Actions verifies the sim, then POSTs the Forge deploy trigger → Forge runs the deploy script (git pull) on the droplet → nginx serves the freshly-pulled static files to visitors at https://rackoon.cloud → Actions curls the live URL to confirm a 200 + a known string.

Planning artifact — review only. Nothing here deploys automatically until the GitHub secrets are set and the workflow is enabled. 🦝
